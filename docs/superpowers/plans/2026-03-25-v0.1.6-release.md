# NexPad v0.1.6 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cut and publish the NexPad v0.1.6 release, shipping UI themes and touchpad scroll tuning.

**Architecture:** Write release notes to `docs/releases/v0.1.6.md`, commit to `main`, push the `v0.1.6` tag — CI handles build, packaging, and attachment to the GitHub Release automatically.

**Tech Stack:** Git, GitHub Actions (build.yml), PowerShell scripts (`build-all.ps1`, `package-release.ps1`, `verify-release-checksums.ps1`), GitHub CLI (`gh`)

**Spec:** `docs/superpowers/specs/2026-03-25-v0.1.6-release-design.md`

### Task 1: Pre-tag verification

**Files:**
- Read: `Windows/NexPad/config.ini`
- Read: `Configs/config_default.ini`
- Read: `README.md`

- [ ] **Step 1: Confirm repo is clean**

```bash
git status
```
Expected: `nothing to commit, working tree clean` on branch `main`. If not, resolve before continuing.

- [ ] **Step 2: Confirm config.ini has the new keys with correct defaults**

Open `Windows/NexPad/config.ini` and verify these keys are present with valid defaults:
```ini
UI_THEME_MODE = 0
TOUCHPAD_SCROLL_ENABLED = 1
TOUCHPAD_SCROLL_SPEED = 1.000
```
`UI_THEME_MODE = 0` is dark (the existing default). `TOUCHPAD_SCROLL_ENABLED = 1` means enabled by default.

- [ ] **Step 3: Confirm Configs/config_default.ini has the same keys**

Open `Configs/config_default.ini` and verify the same three keys are present with the same defaults as above.

- [ ] **Step 4: Confirm README documents the new keys**

Search `README.md` for `UI_THEME_MODE`, `TOUCHPAD_SCROLL_ENABLED`, and `TOUCHPAD_SCROLL_SPEED`. All three should appear in the Configuration section.

### Task 2: Write release notes

**Files:**
- Create: `docs/releases/v0.1.6.md`

- [ ] **Step 1: Create the release notes file**

Create `docs/releases/v0.1.6.md` with exactly this content:

```markdown
NexPad v0.1.6
=================

UI themes and touchpad scroll tuning.

What Changed
======

* Added light and high-contrast UI theme options selectable from the Settings tab
  and via the new UI_THEME_MODE config key (0 = dark, 1 = light, 2 = high contrast).
* Added independent touchpad scroll controls: TOUCHPAD_SCROLL_ENABLED and
  TOUCHPAD_SCROLL_SPEED let you tune or disable DualSense two-finger scroll
  behavior without affecting one-finger cursor movement.

Validation
======

* Verified full solution matrix builds with scripts/build-all.ps1.
* Verified local release packaging with scripts/package-release.ps1 -Version v0.1.6.
* Verified checksum validation with scripts/verify-release-checksums.ps1.

Included Assets
======

* NexPad-win32-v0.1.6.zip
* NexPad-win32-v0.1.6.zip.sha256
* NexPad-x64-v0.1.6.zip
* NexPad-x64-v0.1.6.zip.sha256

Checksum Verification
======

Windows PowerShell example:

```powershell
$expected = (Get-Content .\NexPad-win32-v0.1.6.zip.sha256).Split(' ')[0]
$actual = (Get-FileHash .\NexPad-win32-v0.1.6.zip -Algorithm SHA256).Hash.ToLower()
$actual -eq $expected
```

Repeat the same check for the x64 archive before distribution or use.

Notes
======

This release adds UI theme flexibility for different lighting environments and
display preferences, and gives DualSense touchpad users independent control over
scroll sensitivity without affecting pointer speed.
```

```bash

```
