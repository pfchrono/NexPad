# NexPad v0.1.6.1 Patch Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the v0.1.6 packaging defect where release zips were missing the `presets/` directory, and cut v0.1.6.1.

**Architecture:** Add `presets/.gitkeep` so MSBuild has a source directory to copy, harden `Copy-DirectoryContents` in the packaging script to throw on missing source paths, write release notes, then follow the established notes-first CI-driven release flow.

**Tech Stack:** PowerShell (`scripts/package-release.ps1`), Git, GitHub Actions (CI via `.github/workflows/build.yml`), GitHub CLI (`gh`)

**Spec:** `docs/superpowers/specs/2026-03-25-v0.1.6.1-patch-design.md`

### Task 1: Add presets placeholder and harden packaging script

**Files:**
- Create: `presets/.gitkeep`
- Modify: `scripts/package-release.ps1:81-83`

**Background:** `Copy-DirectoryContents` (line 72) has a guard at lines 81-83 that silently returns when the source path doesn't exist. This caused `presets/` to be silently absent from v0.1.6 zips. All four MSBuild post-build events copy from `$(SolutionDir)..\presets` into `$(OutDir)presets` — adding `.gitkeep` gives them a valid source.

- [ ] **Step 1: Create the presets placeholder**

```bash
mkdir -p F:/code/NexPad/presets
```

Then create an empty file `presets/.gitkeep` (zero bytes, no content).

- [ ] **Step 2: Verify the placeholder exists**

```bash
ls F:/code/NexPad/presets/
```

Expected: `.gitkeep` listed.

- [ ] **Step 3: Read the current Copy-DirectoryContents function**

Open `scripts/package-release.ps1` and locate lines 72-87:

```powershell
function Copy-DirectoryContents {
  param(
    [Parameter(Mandatory = $true)]
    [string]$Source,

    [Parameter(Mandatory = $true)]
    [string]$Destination
  )

  if (!(Test-Path $Source)) {
    return
  }

  New-Item -ItemType Directory -Path $Destination -Force | Out-Null
  Copy-Item (Join-Path $Source "*") $Destination -Recurse -Force
}
```

- [ ] **Step 4: Replace the silent return with a hard error**

Change lines 81-83 from:

```powershell
  if (!(Test-Path $Source)) {
    return
  }
```

To:

```powershell
  if (!(Test-Path $Source)) {
    throw "Copy-DirectoryContents: source path not found: $Source"
  }
```

- [ ] **Step 5: Verify the change is correct**

Read `scripts/package-release.ps1` lines 72-87 and confirm the throw is in place and the function otherwise unchanged.

- [ ] **Step 6: Commit**

```bash
git add presets/.gitkeep scripts/package-release.ps1
git commit -m "fix: add presets placeholder and harden Copy-DirectoryContents"
```

- [ ] **Step 1: Create the release notes file**

Create `docs/releases/v0.1.6.1.md` with exactly this content:

````markdown
NexPad v0.1.6.1
=================

Packaging fix: restore presets directory in release zips.

What Changed
======

* Added presets/ directory to the repository so release zips now include it.
* Fixed package-release.ps1 to error on missing expected source directories
  instead of silently skipping them.

Validation
======

* Verified full solution matrix builds with scripts/build-all.ps1.
* Verified local release packaging with scripts/package-release.ps1 -Version v0.1.6.1.
* Verified release zip contains NexPad.exe, config.ini, presets/, README.md, LICENSE.
* Verified checksum validation with scripts/verify-release-checksums.ps1.

Included Assets
======

* NexPad-win32-v0.1.6.1.zip
* NexPad-win32-v0.1.6.1.zip.sha256
* NexPad-x64-v0.1.6.1.zip
* NexPad-x64-v0.1.6.1.zip.sha256

Checksum Verification
======

Windows PowerShell example:

```powershell
$expected = (Get-Content .\NexPad-win32-v0.1.6.1.zip.sha256).Split(' ')[0]
$actual = (Get-FileHash .\NexPad-win32-v0.1.6.1.zip -Algorithm SHA256).Hash.ToLower()
$actual -eq $expected
```

Repeat the same check for the x64 archive before distribution or use.

Notes
======

This patch corrects a packaging oversight in v0.1.6 where the presets/ directory
was absent from release zips. No feature changes from v0.1.6.
````
