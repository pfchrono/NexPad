# NexPad v0.1.6 Release Design

**Date:** 2026-03-25
**Version:** v0.1.6
**Approach:** Notes-first, then tag (CI-driven)

## What Ships

Two features merged to `main` since v0.1.5:

- **UI themes** (issue #6): Light and high-contrast theme options, selectable from the Settings tab or via `UI_THEME_MODE` config key (`0` = dark, `1` = light, `2` = high contrast).
- **Touchpad scroll controls** (issue #7): Independent `TOUCHPAD_SCROLL_ENABLED` and `TOUCHPAD_SCROLL_SPEED` settings for DualSense two-finger scroll behavior, decoupled from one-finger cursor settings.

No controller parsing, transport path, or HID changes are included. Hardware regression matrix scope is minimal.

## Release Notes Content

File: `docs/releases/v0.1.6.md`

```
NexPad v0.1.6
=================

UI themes and touchpad scroll tuning.

What Changed
======

* Added light and high-contrast UI theme options selectable from the Settings tab
  and via the new UI_THEME_MODE config key (0 = dark, 1 = light, 2 = high contrast).
* Added independent touchpad scroll controls: TOUCHPAD_SCROLL_ENABLED and
  TOUCHPAD_SCROLL_SPEED let you tune or disable DualSense two-finger scroll
  behavior without affecting one-finger cursor movement.

Validation
======

* Verified full solution matrix builds with scripts/build-all.ps1.
* Verified local release packaging with scripts/package-release.ps1 -Version v0.1.6.
* Verified checksum validation with scripts/verify-release-checksums.ps1.

Included Assets
======

* NexPad-win32-v0.1.6.zip
* NexPad-win32-v0.1.6.zip.sha256
* NexPad-x64-v0.1.6.zip
* NexPad-x64-v0.1.6.zip.sha256

Checksum Verification
======

Windows PowerShell example:

```powershell
$expected = (Get-Content .\NexPad-win32-v0.1.6.zip.sha256).Split(' ')[0]
$actual = (Get-FileHash .\NexPad-win32-v0.1.6.zip -Algorithm SHA256).Hash.ToLower()
$actual -eq $expected
```

Repeat the same check for the x64 archive before distribution or use.

Notes
======

This release adds UI theme flexibility for different lighting environments and
display preferences, and gives DualSense touchpad users independent control over
scroll sensitivity without affecting pointer speed.
```

## Execution Steps

**Before tagging:**
1. Confirm `main` is clean (no uncommitted changes).
2. Confirm `config.ini` and `Configs/config_default.ini` include correct defaults for the new keys: `UI_THEME_MODE`, `TOUCHPAD_SCROLL_ENABLED`, `TOUCHPAD_SCROLL_SPEED`.
3. Verify README documents the new keys (expected: already done, but confirm).

**Release:**
4. Write `docs/releases/v0.1.6.md` from the content above.
5. Commit to `main`: `docs: add v0.1.6 release notes`
6. Push `main` and confirm CI passes.
7. Push tag: `git tag v0.1.6 && git push origin v0.1.6`
8. CI builds Win32 and x64, packages zips and SHA256s, attaches to GitHub Release.
9. Draft and publish GitHub Release using the committed release notes.

**After release:**
10. Verify the GitHub Release page renders correctly.
11. Confirm packaged zip files download successfully.
12. Capture any follow-up fixes for the next patch release.

## Out of Scope

- No README edits — new config keys already documented.
- No source version bumps — version injected at build time from the git tag via `NexPadVersion` MSBuild property.
- No controller hardware validation matrix pass required — no input path changes.
