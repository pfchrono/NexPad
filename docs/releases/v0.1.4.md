NexPad v0.1.4
=================

DualSense gesture and status-surface release with two-finger scrolling, battery bar UI, and in-app version display.

What Changed
======

* Added DualSense two-finger touchpad scrolling with safe gesture arbitration alongside the existing one-finger cursor and tap behavior.
* Preserved fail-safe behavior across disconnects, transport changes, and limited Bluetooth touch-report paths.
* Updated generated and shipped touchpad setting descriptions to document one-finger cursor and two-finger scroll behavior.
* Replaced the status-page battery text-only row with a color-coded battery bar that keeps charging and fallback states visible.
* Added a bottom-right in-app version label in `v-<version>` format.
* Wired package and build version flow through a shared app-version source so packaged artifact names and the in-app version label stay aligned.

Validation
======

* Verified local solution builds with `scripts/build-all.ps1`.
* Verified `Debug|x64` builds cleanly after the final status UI changes.
* Verified local release packaging produces both zip archives and matching `.sha256` files for `v0.1.4`.
* Verified the checksum verification script passes against the generated `v0.1.4` artifact set.
* Manually validated the status-page battery bar visuals and version footer.

Included Assets

Checksum Verification
======

Windows PowerShell example:

```powershell
$expected = (Get-Content .\NexPad-win32-v0.1.4.zip.sha256).Split(' ')[0]
$actual = (Get-FileHash .\NexPad-win32-v0.1.4.zip -Algorithm SHA256).Hash.ToLower()
$actual -eq $expected
```

Repeat the same check for the x64 archive before distribution or use.

Notes
======

This release keeps the existing controller mapping behavior intact while improving DualSense touch support and the main status page. The version shown in the UI is sourced at build time and falls back cleanly for local builds.
