NexPad v0.1.5
=================

Startup workflow and release trust-signal update.

What Changed
======

* Added a new **Start with Windows** setting in the native Settings tab so users can enable or disable startup behavior from inside NexPad.
* Wired startup behavior through persisted config (`START_WITH_WINDOWS`) and current-user Windows startup registration.
* Added Windows executable file version metadata so file properties stay aligned with the shared build/release version flow.
* Added a dedicated controller compatibility and troubleshooting guide:
  * compatibility matrix by controller family and transport path
  * common detection and transport troubleshooting steps
  * reusable release hardware-validation matrix
* Updated release workflow docs to reference the compatibility matrix during release validation.
* Refreshed README screenshots to reflect the current UI.

Validation
======

* Verified full solution matrix builds with `scripts/build-all.ps1`.
* Verified local screenshot capture completes with `scripts/capture-screenshots.ps1` against `release/x64/NexPad.exe`.
* Verified local release packaging with `scripts/package-release.ps1 -Version v0.1.5`.
* Verified checksum validation with `scripts/verify-release-checksums.ps1`.

Included Assets
======

* `NexPad-win32-v0.1.5.zip`
* `NexPad-win32-v0.1.5.zip.sha256`
* `NexPad-x64-v0.1.5.zip`
* `NexPad-x64-v0.1.5.zip.sha256`

Checksum Verification
======

Windows PowerShell example:

```powershell
$expected = (Get-Content .\NexPad-win32-v0.1.5.zip.sha256).Split(' ')[0]
$actual = (Get-FileHash .\NexPad-win32-v0.1.5.zip -Algorithm SHA256).Hash.ToLower()
$actual -eq $expected
```

Repeat the same check for the x64 archive before distribution or use.

Notes
======

This release improves first-run and living-room usability by moving startup configuration into the app itself, while also improving release confidence through executable metadata alignment and clearer compatibility/troubleshooting documentation.
