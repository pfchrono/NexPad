NexPad v0.1.1
=================

Focused polish release for stability, branding, and local build tooling.

What Changed
======

* Fixed a scroll-redraw distortion bug in the explanatory status text box by treating read-only multiline edit controls as opaque edit surfaces during Windows color handling.
* Replaced the NexPad application icon with a newly designed icon and added the source assets for maintaining it.
* Added `docs/assets/nexpad-icon-source.svg` as the editable source art for the new icon.
* Added `docs/assets/nexpad-icon-preview.png` as a generated preview of the current icon design.
* Added `scripts/generate-icon.ps1` so the icon can be regenerated reproducibly.
* Added `scripts/build-all.bat` to build the full Visual Studio solution matrix across Debug and Release for Win32 and x64.
* Extended `scripts/build-all.bat` with an optional clean mode using `clean`, `--clean`, or `/clean`.
* Added `scripts/build-all.ps1` as a matching PowerShell wrapper for the same build matrix, including optional `-Clean` support.
* Updated the README build documentation to include the new batch and PowerShell helper scripts.

Validation
======

* Rebuilt the solution successfully for the full supported matrix:
  * `Debug|Win32`
  * `Debug|x64`
  * `Release|Win32`
  * `Release|x64`
* Revalidated `Debug|Win32` directly after the matrix run.

Included Assets
======

* `NexPad-win32-v0.1.1.zip`
* `NexPad-x64-v0.1.1.zip`

Checksum Verification
======

Published SHA256 checksum files were introduced in `v0.1.2`.

This release does not include `.sha256` assets. For checksum-verified packaged downloads, use `v0.1.2` or later.

Notes
======

This release keeps the feature surface stable while improving the app presentation and the local developer workflow.
