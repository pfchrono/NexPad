NexPad v0.1.6.1
=================

Packaging fix: restore presets directory in release zips.

What Changed
======

* Added presets/ directory to the repository so release zips now include it.
* Fixed package-release.ps1 to error on missing expected source directories
  instead of silently skipping them.

Validation
======

* Verified full solution matrix builds with scripts/build-all.ps1.
* Verified local release packaging with scripts/package-release.ps1 -Version v0.1.6.1.
* Verified release zip contains NexPad.exe, config.ini, presets/, README.md, LICENSE.
* Verified checksum validation with scripts/verify-release-checksums.ps1.

Included Assets
======

* NexPad-win32-v0.1.6.1.zip
* NexPad-win32-v0.1.6.1.zip.sha256
* NexPad-x64-v0.1.6.1.zip
* NexPad-x64-v0.1.6.1.zip.sha256

Checksum Verification
======

Windows PowerShell example:

```powershell
$expected = (Get-Content .\NexPad-win32-v0.1.6.1.zip.sha256).Split(' ')[0]
$actual = (Get-FileHash .\NexPad-win32-v0.1.6.1.zip -Algorithm SHA256).Hash.ToLower()
$actual -eq $expected
```

Repeat the same check for the x64 archive before distribution or use.

Notes
======

This patch corrects a packaging oversight in v0.1.6 where the presets/ directory
was absent from release zips. No feature changes from v0.1.6.
