NexPad v0.1.3
=================

DualSense touchpad release with live settings support and default-on startup behavior.

What Changed
======

* Added first-pass DualSense touchpad cursor movement through the native HID fallback path.
* Corrected the DualSense touch packet decoding after hardware validation so vertical and horizontal touchpad movement behave correctly.
* Added live Settings-tab controls for `TOUCHPAD_ENABLED` and `TOUCHPAD_SPEED`.
* Enabled DualSense touchpad cursor support by default in generated and shipped config files.
* Added config support for `TOUCHPAD_ENABLED`, `TOUCHPAD_DEAD_ZONE`, and `TOUCHPAD_SPEED` across config load, save, and preset profile import/export.
* Updated the README to document DualSense touchpad support and the new Settings-tab controls.

Validation
======

* Verified the DualSense touchpad implementation builds cleanly in `Debug|Win32`.
* Verified the Settings-tab touchpad controls build cleanly in `Debug|Win32`.
* Verified local release packaging produces both zip archives and matching `.sha256` files for `v0.1.3`.
* Verified the checksum verification script passes against the generated `v0.1.3` artifact set.
* Hardware-validated the DualSense touchpad behavior: input registers correctly and the current movement values behave well.

Expected Included Assets
======

* `NexPad-win32-v0.1.3.zip`
* `NexPad-win32-v0.1.3.zip.sha256`
* `NexPad-x64-v0.1.3.zip`
* `NexPad-x64-v0.1.3.zip.sha256`

Checksum Verification
======

Windows PowerShell example:

```powershell
$expected = (Get-Content .\NexPad-win32-v0.1.3.zip.sha256).Split(' ')[0]
$actual = (Get-FileHash .\NexPad-win32-v0.1.3.zip -Algorithm SHA256).Hash.ToLower()
$actual -eq $expected
```

Repeat the same check for the x64 archive before distribution or use.

Notes
======

This first touchpad release is DualSense-only and movement-only. Stick-based mouse control remains available, and touchpad movement is added as an additional cursor input source rather than replacing the stick pipeline.
